## Base URL

All Zephyra API requests should be sent to the following base URL:

```bash
https://api.zephyra.dev/v1
```

---

## Authentication

Zephyra APIs use **Firebase Authentication** with Bearer tokens.

<Columns cols={2}>
  <Card title="Client Side" icon="user">
    The frontend (e.g., Nuxt, React, or mobile app) signs in a user using Firebase SDK and retrieves an **ID Token**.
  </Card>

  <Card title="Server Side" icon="server">
    The backend verifies the token via **Firebase Admin SDK** before granting access to protected routes.
  </Card>
</Columns>

Each request to a protected endpoint must include the token in the header:

```bash
Authorization: Bearer <YOUR_ID_TOKEN>
```

---

## Response Format

All API responses follow a consistent structure:

```json
{
  "success": true,
  "data": {},
  "message": "Operation successful"
}
```

| Field     | Type    | Description                                  |
| --------- | ------- | -------------------------------------------- |
| `success` | boolean | Indicates if the request was successful      |
| `data`    | object  | Contains the returned payload or record data |
| `message` | string  | A short message describing the result        |

---

## Error Handling

Error responses follow a unified format as well:

```json
{
  "success": false,
  "message": "Invalid or expired token"
}
```

Common error codes:

| Code | Description                             |
| ---- | --------------------------------------- |
| 400  | Bad Request — Invalid parameters        |
| 401  | Unauthorized — Missing or invalid token |
| 403  | Forbidden — Insufficient permissions    |
| 404  | Not Found — Resource doesn’t exist      |
| 500  | Internal Server Error                   |
